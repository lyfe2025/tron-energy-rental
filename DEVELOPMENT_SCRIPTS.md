# 开发脚本说明

## 根目录脚本文件

以下是保留在根目录的开发脚本文件及其用途：

### 🔄 restart.sh
**简化重启脚本**
- 用途：快速重启前后端服务
- 功能：停止现有进程，重新启动开发服务
- 使用：`./restart.sh`

### 🌐 ngrok.sh
**ngrok管理脚本**
- 用途：快速启动ngrok内网穿透服务
- 功能：调用scripts/development/ngrok-manager.sh
- 使用：`./ngrok.sh`

### 🛠️ project.sh
**项目统一管理脚本**
- 用途：项目的完整管理入口
- 功能：服务启动/停止/重启、日志管理、数据库管理
- 版本：v2.1 (安全增强版)
- 使用：`./project.sh` (交互式菜单)

## 推荐使用优先级

1. **开发重启**：`npm run restart` 或 `pnpm restart` 
2. **快速重启**：`./restart.sh`
3. **完整管理**：`./project.sh`
4. **内网穿透**：`./ngrok.sh`

## 注意事项

- 所有脚本都有执行权限
- 主要脚本逻辑在 `scripts/` 目录下的模块化文件中
- 这些根目录脚本主要作为快捷入口使用
