登录API测试

问题分析

页面被刷新的根本原因：

  1. 登录请求返回401状态码：这是不正常的，因为用户还没有认证
  2. API响应拦截器触发：401状态码触发了window.location跳转
  3. 页面刷新：window.location导致真正的页面刷新

API测试

测试登录API的响应，确认是否返回了错误的401状态码

修复方案

✅ 已修复：移除window.location

在API响应拦截器中完全移除了window.location的使用，改为触发自定义事件

✅ 已修复：全局事件处理

在App.vue中添加了全局的token过期事件监听器，使用Vue Router进行跳转

⚠️ 需要检查：后端API逻辑

登录请求不应该返回401状态码，需要检查后端API的实现

调试步骤

  1. 运行API测试：使用上面的表单测试登录API
  2. 检查响应状态码：确认是否返回401
  3. 检查后端日志：查看后端API的处理逻辑
  4. 验证修复效果：确认页面不再刷新
